JavaEE - Tomcat
发表于2020-08-18|更新于2020-09-16|JavaJavaEE
|字数总计:1.5k|阅读时长:4分钟|阅读量:|评论数:

web 相关概念回顾：

软件架构：

  • C/S： 客户端 / 服务器端
  • B/S： 浏览器 / 服务器端

资源分类：

  • 静态资源： 可以理解为前端的固定页面，这里面包含 HTML、CSS、JS、图片等等，不需要查数据库也不需要程序处理，直接就能够显示的页面，如果想修改内容则必须修改页面，但是访问效率相当高。
  • 动态资源： 需要程序处理或者从数据库中读数据，能够根据不同的条件在页面显示不同的数据，内容更新不需要修改页面但是访问速度不及静态页面。
  • 判断依据： 是否与数据库进行交互

网络通信三要素：

  1. IP： 电子设备(计算机)在网络中的唯一标识。
  2. 端口： 应用程序在计算机中的唯一标识。[0~65536]
  3. 传输协议： 规定了数据传输的规则
    • tcp： 安全协议，三次握手。 速度稍慢
    • udp： 不安全协议。 速度快

一、服务器相关概述

服务器： 安装了服务器软件的计算机

服务器软件： 接收用户的请求，处理请求，做出响应

web 服务器软件： 接收用户的请求，处理请求，做出响应。在 web 服务器软件中，可以部署 web 项目，让用户通过浏览器来访问这些项目，所以又被称为 web 容器

JavaEE： Java 语言在企业级开发中使用的技术规范的总和，一共规定了 13 项大的规范

常见的 java 相关的 web 服务器软件：

  • webLogic： Oracle 公司，大型的 JavaEE 服务器，支持所有的 JavaEE 规范，收费的。
  • webSphere： IBM公司，大型的 JavaEE 服务器，支持所有的 JavaEE 规范，收费的。
  • JBOSS： JBOSS 公司的，大型的 JavaEE 服务器，支持所有的 JavaEE 规范，收费的。
  • Tomcat： Apache 基金组织，中小型的 JavaEE 服务器，仅支持少量的 JavaEE 规范 (servlet / jsp)， 开源免费

TomCat 版本与 Java 版本的对应关系：

1：Tomcat 的下载与安装

安装 Tomcat 之前，必须存在 Java 环境 (安装了 jdk)

下载： 官网 下载链接

安装： 解压压缩包即可，注意：安装目录建议不要有中文和空格

卸载： 删除目录就行了

目录结构：

重要的配置文件：

  • 核心配置文件：server.xml
  • 所有 web 项目默认配置文件：web.xml
  • 用户权限配置文件：tomcat-users.xml

2：Tomcat 的启动与关闭

启动： 双击 bin 目录下的 startup.bat 即可

访问： 浏览器输入 http://localhost:8080 之后可以访问本地的 Tomcat。要访问别人的 Tomcat 将 localhost 换成别人的 IP 即可

可能遇到的问题：

  • 黑窗口一闪而过： 原因： 没有正确配置 JAVA\_HOME 环境变量

  • 端口占用：

    方式一： 找到占用的端口号，并且找到对应的进程，杀死该进程 netstat -ano

    方式一： 修改自身的端口号，找到 conf 目录下的 server.xml

关闭：

  • 正常关闭： 双击 bin 目录下 的 shutdown.bat ，或者 Ctrl + C

  • 强制关闭： 点击启动窗口的 ×

二：项目的部署方式

方式一：

  1. 直接将项目放到 webapps 目录下即可。

    /hello：项目的访问路径 → 虚拟目录

  2. 简化部署：将项目打成一个 war 包，再将 war 包放到 webapps 目录下。war 包会自动解压缩

方式二：

  1. 配置 conf 目录下的 server.xml 文件

  2. 在 Host 标签体中配置

    docBase： 项目存放的路径

    path： 虚拟目录

方式三：热部署

  1. 在 conf\Catalina\localhost 创建任意名称的 xml 文件。

  2. 在文件中编写

    虚拟目录： xml 文件的名称

java 动态项目的目录结构，层级关系如下：

  • 项目的根目录
    • WEB-INF目录： 不能被外界直接访问
      • web.xml： web 项目的核心配置文件
      • classes 目录： 放置字节码文件的目录
      • lib 目录： 放置依赖的 jar 包

三、IDEA 中使用 Tomcat

1：Tomcat 集成到 IDEA

  1. 点击 Configure 选择 Settings

  2. 找到 Build...... ment 下的 Application Servers，点小 + 号，选择 TomcatServer

2：IDEA 创建 JavaWeb 项目

热加载： 即时更新，点击 菜单栏 → Run → Edit Configurations → Update classes and resources 或者 Update resources。启动时选择 debug 方式启动

3：Tomcat 日志控制台乱码问题

如果出现以下情况

出现原因： 中国区域 windows 默认使用 gbk 编码，IDEA 控制台跟着也是使用的 gbk，而 tomcat 的日志输出控制台编码为 utf-8

所以解决方式有两种：

  1. 修改整个 windows 控制台的编码为 utf-8

    管理员身份运行 cmd 输入命令： chcp 65001 切换 cmd 为 utf8。切换回 gbk 可输入：chcp 936

    这种方式是临时的，要彻底修改需要修改注册表，所以也不推荐

  2. 修改 tomcat 日志控制台的编码为 gbk

    tomcat 日志控制台的配置文件在 tomcat安装目录\conf\logging.properties 但是修改该配置文件可能会导致 get / post参数乱码，所以这种方式不推荐

最终解决方案如下：

  1. 强制修改 IDEA 控制台编码为 utf-8

    在 IDEA工具栏中点击 → Help → custom vm options 添加 -Dfile.encoding=UTF-8，强制为 utf8 编码显示。

    不要自己改去修改安装目录下的 idea64.exe.vmoptions 配置文件，因为 idea 一般会将该配置文件向用户目录复制一个并优先使用

  2. 使用 System.out.println 打印时控制台再次显示乱码

    需在添加 Tomcat 服务器时 VM 选项加上参数 -Dfile.encoding=utf-8
