一、Servlet 概述

1：简介

Servlet： Server Applet，运行在服务器端的小程序，主要是用来处理用户请求的

狭义上讲是 javax.servlet.Servlet 接口及其子接口，广义上讲：实现了 Servlet 接口的实现类我们都称为 Servlet

将来我们自定义一个类，实现 Servlet 接口，复写方法。Servlet 对象由 Servlet 容器（如 Tomcat）帮我们创建，不需要我们 new

2：快速入门

  1. 创建 JavaEE 项目

  2. 定义一个类，实现 Servlet 接口，实现接口中的抽象方法

  3. 在 web.xml 中注册 Servlet

二、Servlet 执行原理

  1. 当服务器接受到客户端浏览器的请求后，会解析请求 URL 路径，获取访问的 Servlet 的资源路径

  2. 查找 web.xml 文件，是否有对应的 <url-pattern> 标签体内容。

  3. 如果有，则根据 <servlet-name> 找到对应的 <servlet-class> 全类名

  4. tomcat 会将字节码文件加载进内存

  5. 通过反射创建其对象

  6. 调用其方法 (先调用 init 方法)

1：被创建之后，调用 init 方法，只执行一次

Servlet 什么时候被创建？默认情况下，第一次被访问时，Servlet 被创建。可以配置执行 Servlet 的创建时机，在 <servlet> 标签下配置 <load-on-startup> 标签

  • 第一次被访问时创建，配置值为负数

  • 在服务器启动时创建，配置值为 0 或正整数

Servlet 对象创建后会调用 init() 方法，只执行一次，说明一个 Servlet 在内存中只存在一个对象，是单例的

多个用户同时访问时，可能存在线程安全问题。

解决：尽量不要在 Servlet 中定义成员变量。即使定义了成员变量，也修改值

2：提供服务：service 方法，执行多次

每次访问 Servlet 时，service() 方法都会被调用一次。

3：被销毁：destroy 方法，只执行一次

服务器关闭时，Servlet 被销毁，只有服务器正常关闭时，才会执行 destroy() 方法。

destroy() 方法在 Servlet 被销毁之前执行，一般用于释放资源

六、Servlet 继承体系

1：GenericServlet

将 Servlet 接口中其他的方法做了默认空实现，只将 service() 方法作为抽象

2：HttpServlet – 常用

对 http 协议的一种封装，它重写并重载了 service() 方法

重写的 service() 方法：

在该方法中先将 ServletRequest 和 ServletResponse 强转为了 HttpServletRequest 和 HttpServletResponse

然后调用重载的 service() 方法，并将刚刚强转得到对象传递到重载的方法中。

重载的 service() 方法：

在方法中获取请求的方式（get 或 post ）

再根据不同的请求方式去调用不同的方法：

  • 如果是 get 请求，则调用 doGet()

  • 如果是 post 请求，则调用 doPost()

结论： 当通过继承 HttpServlet 来创建一个 Servlet 时，我们只需要根据要处理的请求的类型，来重写不同的方法。 处理 get 请求，则重写 doGet() ，处理 post 请求，则重写 doPost()
