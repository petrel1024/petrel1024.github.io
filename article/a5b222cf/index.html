一、IDEA 简介

IDEA：全称 IntelliJ IDEA，是 Java 语言的集成开发环境，在业界被公认为最好的 java 开发工具之一。尤其在智能代码助手、代码自动提示、重构、J2EE 支持、Ant、JUnit、CVS 整合、代码审查、创新的 GUI 设计等方面的功能可以说是超常的

官网： www.jetbrains.com

IDEA 分为两个版本，旗舰版 (Ultimate) 和 社区版 (Community)。旗舰版当然是收费的啦，当然功能也比较强大

建议：tb 买个教育邮箱，然后去官网注册个账号，进行学生认证即可

更多姿势可以参考：

  • 官方文档：https://www.jetbrains.com/help/idea/discover-intellij-idea.html
  • w3cschool 文档：https://www.w3cschool.cn/intellij_idea_doc/

1：下载

官网：https://www.jetbrains.com/idea/ 点击 Download

IDEA 每年会发布三个版本，如：2018.1、2018.2、2018.3，每个版本都会有子版本，如 2018.3.1、2018.3.2

个人建议使用每一个版本的最终版本即可，如 2018.3.6。不建议直接使用每个版本的初始版本，如 2018.3.1，bug 比较多。

注：本文以 Windows 2018 版本为例，相关设置更新会有标注 (如 2020 版本中的设置不一样的地方)

选择对应平台直接下载旗舰版

3：目录结构

安装目录结构：

bin 目录下

可以编辑配置文件，设置VM的参数

自 2019 版本起，直接修改安装目录下的配置文件将不会生效

需要在 IDEA 中点击 help → Edit Custom VMoptions，对该文件进行修改

配置目录：

C:\Users\${user.name}\.IntelliJIdea2018.3

config → C:\Users\${user.name}\AppData\Roaming\JetBrains\IntelliJIdea2020.1

system → C:\Users\${user.name}\AppData\Local\JetBrains\IntelliJIdea2020.1

这是 IDEA 的各种配置的保存目录。这个设置目录有一个特性，就是你删除掉整个目录之后，重新启动 IntelliJ IDEA 会再自动帮你生成一个全新的默认配置， 所以很多时候如果你把 IntelliJ IDEA 配置改坏了，没关系，删掉该目录，一切都会还原到默认。

只删除 system 文件夹即可，删除 config 之后配置就没了

config： 主要记录了IDE 主要配置功能，自定义的代码模板、文件模板、快捷键等，Project 的 tasks 记录等等个性化的设置

system： IntelliJ IDEA 系统文件目录，里面主要有：缓存、索引、容器文件输出等等

如需修改配置目录位置：

  • 打开 IDEA安装目录\bin\idea.properties

  • 将以下两行的注释放开，并修改值为你要存放的目录，注意写路径时使用正斜杠 /

    idea.config.path=${user.home}/.IntelliJIdea/config

    idea.system.path=${user.home}/.IntelliJIdea/system

1：进入设置

要想对整个 IDEA 进行设置而不是对某个项目进行设置，最好使用下边的方式

不要在创建项目之后再设置 settings 那样只是对该项目起作用，或者可以选择 Other Settings → Settings for new Project（没有创建项目请忽略，使用上一种方式）

导入/导出设置： 做好备份是非常必要滴 (import：导入，Export：导出)

2：必改配置项

2.1：设置文件编码

Transparent native-to-ascii conversion 主要用于转换 ascii，一般都要勾选， 不然 Properties 文件中的注释显示的都不会是中文。

打开文件时设置源文件的编码： IDEA 右下角

点击之后选择更换，若弹窗提示是否更换源文件编码，参考：

  • Reload：仅预览，不修改源文件编码
  • Convert：预览并修改源文件编码

2.2：设置自动编译

Intellij Idea 默认不会将你修改后的 java 文件编译成 class 文件，修改完成需要手动编译，可以设置自动编译

2.3：设置自动导包

Add unambiguous imports on the fly：自动优化导入的包

Optimize imports on the fly：自动去掉一些没有用到的包

2.4：忽略大小写提示

默认情况下，你输入 str 它是不会给你提示出来 String 的，必须输入 Str 才行

3.10：误删文件恢复

IDEA 会记录文件的操作记录，所以即使误删亦可回复

右键误删文件所在项目 → 找到 local history → show history → 找到误删的文件恢复即可

3.11：开启注解处理

使用 Lombok 时记得开

Build → Compiler → Annotation Processors → 勾选 Enable Annotation Processors

3.12：开启 Java 中的面包屑

Editor → General → Breadcrumbs，勾选 Java

3.13：隐藏 .idea 和 .iml

.idea 和 .iml 是 IDEA 自动创建的文件夹 / 文件，若是嫌它看着烦可以隐藏

Edit → File Types，在 Ignore files and folders 中添加 .idea;*.iml;

3.14：不打开最近使用的项目

启动 IDEA 之后，默认打开最后一次使用的项目，可以关闭该行为，视个人喜好而定

Appearance&Behavior → System Settings，取消 Reopen last projects on startup 的勾

3.15：将自定义单词添加到字典

IDEA 会对单词进行拼写检查，关闭了不合适，有时拼写错了它可以告诉你。但是不关，一些第三方框架起的名字它检测不到，总是给你显示个黄色的波浪线，看着烦人。

可以将诸如：dubbo、nacos 等这里单词添加到 IDEA 的字典中

在拼写警告的单词上按快捷键 Alt + Enter，选择添加到字典

3.16：关闭使用 MyBatis 时 SQL 语句下的警告线

使用 MyBatis 框架时，Mapper 映射配置文件中写的 SQL 语句会有一堆黄色的警告线，看起来比较烦人

报黄主要是有两项警告：

  1. 没有配置数据库
  2. SQL 方言检测的检查提示项

关闭步骤：

  1. File → Setting → Editor → Inspections，找到 SQL 选项并展开

  2. 将 No data sources configured、SQL dialect detection这两项的勾选去掉，点击 ok

3：安装 Jrebel 热加载插件

据说你修改完 Java 代码还在重启容器？启动要多久，不小心少写了一个字符又要重启？累不累。

使用 Jrebel 可以在不重启容器的情况下进行热加载

为什么不是 springboot-devtools？

  • 首先明确一点，引入这些工具只是方便开发，节省时间的。引入 springboot-devtools 的依赖，生产环境是不需要，开发者引入后容易误提交到线上。
  • springboot-devtools 方式的热部署在功能上有限制。如在方法内的修改可以实现热部署，但是新增的方法或者修改方法参数之后热部署是不生效的。
  • 相对于 JRebel，springboot-devtools 方式热部署的速度可能有点慢

安装： 搜索 JRebel and XRebel for IntelliJ 安装即可

JRebel：热加载插件；XRebel：性能分析插件。新版本将两个整合在一起了

破解： 该插件是收费的，所以需要破解，需要的东西如下：

注意请支持正版，本教程仅支持个人学习使用，请不要用于商业用途！

    • https://www.guidgen.com/

五、模板

1：预定义模板

模板有的只能单独使用 (如：psvm)，大部分既可以单独使用还可以用于后缀补全

使用和快速设置模板快捷键：

  • 插入代码模板：Ctrl + J，该快捷键亦可用于后缀补全
  • 将选定的代码加入动态模板中：Ctrl + Alt + J

常用代码模板：

  • ! + Tab：快速生成 html 模板

  • psvm： 生成 main 方法

  • sout：快速打印，类似的还有：

    • soutp = System.out.println("方法形参名 = " + 形参名);
    • soutv = System.out.println("变量名 = " + 变量);
    • soutm = System.out.println("当前类名.当前方法");

  • for：foreach 遍历，也可以使用 iter

  • fori：带索引的遍历，也可以使用 itar

  • var：声明变量

  • null：判断对象为空

  • notnull：判断对象非空

  • nn：判断对象非空

  • not：对布尔类型取反

  • if：条件判断

  • cast：强转对象 / 数据类型

  • return：返回值
